# DWDP: 去除层间同步的分布式权重数据并行推理

> 各 rank 完全 DP 独立执行，MoE 专家权重分散在 peer GPU，运行时按需异步 P2P 拉取，**彻底消灭 MoE 层间 a2a 同步屏障**。

## 基本信息

- **arXiv**：2604.01621
- **硬件目标**：GB200 NVL72（需要 ≥ 1.8 TB/s per GPU P2P 带宽）
- **模型目标**：DeepSeek-R1（NVFP4 MoE）
- **消化日期**：2026-04-22

## 核心观点

1. **a2a 是 MoE 推理对不均衡敏感的主要诱因**：实测 DEP 策略下同步开销占 12%（20% CV 负载偏斜时）。
2. **DWDP 的破局点**：保持各 rank 完全独立执行，MoE 专家权重跨 GPU 分布存储，按需异步 P2P 预取。
3. **硬件前提**：需要 NVLink 级 P2P 带宽，才能在一层计算窗口内完成下一层权重预取。
4. **工程优化一**：扩展 groupedGEMM 接受 TensorList 多缓冲区输入，消除 D2D 合并拷贝（+3% TPS/GPU）。
5. **工程优化二**：时分复用（TDM）缓解多对一通信竞争，在计算窗口小的场景下是必需。
6. **端到端加速**：20-100 TPS/user 范围内 TPS/GPU +8.8%（TTFT 有牺牲）。

## 与其他素材的关联

- 与 [[MegaScale-Infer]]：都是 MoE 推理的 a2a 优化，MegaScale-Infer 保留 a2a 但用 ping-pong pipeline 隐藏，DWDP 直接消灭 a2a。
- 与 [[ChunkFlow]]：docx 原笔记明确讨论："解决负载均衡的有效方法是优化掉 a2a 操作，那么 CP 中的 a2a 如何被优化掉，改成 ChunkFlow 那样的类串行？" —— 这是跨素材的核心洞察。
- 与 [[GB200-NVL72]]：DWDP 成立的硬件基础。1.8 TB/s per GPU 的 NVLink 5.0 + 72 GPU 单 NVLink 域是必要条件。
- 与 [[AFD-Ratio 理论]]：同是推理解耦方向，但 AFD 解耦 Attention/FFN，DWDP 保留耦合但分散 expert 权重。

## 原文精彩摘录

> 推理中的 a2a 操作使得其对不均衡现象十分敏感，DWDP 尝试完全不进行 a2a 操作，仅使用 p2p 做远程权重 prefetch。

> 要求计算复杂度能 overlap 掉远程权重 prefetch，一个关键要求是高带宽通信。像 NVL72 这样的系统能够在域内的任意两个 GPU 之间提供高带宽通信，从而可以将远程权重预取隐藏在计算之后。

## 适用边界

- 适用：**MoE 推理**（专家权重占显存主体），context server 场景。
- 不适用：decode 阶段（计算窗口太小），训练 backward（梯度需要同步回传）。<!-- confidence: INFERRED -->
