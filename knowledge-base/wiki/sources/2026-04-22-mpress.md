# MPress: 单节点多 GPU 亿级参数模型训练

> 以流水线并行为基础，利用 D2D NVLink swap 把闲置阶段 GPU 的显存变成活跃阶段的"虚拟扩展"，避免昂贵的 PCIe offload。

## 核心观点

1. **GPU 显存墙的三条传统路径都有硬伤**：CPU swap 受 PCIe 带宽压制、激活重计算开销大、ZeRO 通信量高。
2. **流水线并行天然存在"闲置显存"**：同一时刻活跃 stage 显存紧张，其他 stage 显存有余 → 可以跨 GPU 借用。
3. **D2D swap（NVLink, ~600 GB/s）远快于 CPU swap（PCIe, ~16 GB/s）**，把 offload 的目标从 CPU 换成同节点其他 GPU 是核心创新。
4. **三级联合优化**：流水线并行 + 选择性重计算 + D2D swap，由静态 plan 和 runtime executor 联合调度。
5. **关键限定**：单节点场景。方案前提是"GPU 间对称拓扑 + NVLink 互联"。

## 与其他素材的关联

- 与 [[DWDP]] 的关系：二者都利用 NVLink 做 GPU 间传输避开 PCIe，但 MPress 是**训练显存扩展**（swap），DWDP 是**推理权重预取**（prefetch），应用场景正交。
- 与 [[ChunkFlow]] 的关系：都在解决显存问题。MPress 用"空间换显存"（借用其他 GPU），ChunkFlow 用"时间换显存"（chunk 串行 + 两次 forward）。
- 与 [[长上下文&超节点调研]] 的关系：调研文档把 MPress 归类为"不同 pipeline stage Device Memory 不均衡"解决方案的代表。
- **待讨论**：静态 plan 在动态场景（MoE 路由）下的局限。<!-- confidence: INFERRED -->

## 原文精彩摘录

> 流水线并行中各阶段并非同时全满——当前活跃阶段显存压力大，而其他阶段 GPU 处于相对空闲状态，其显存可临时借用。

> MPress 的思路不是重新打破计算均衡去追求 memory-balanced partition，而是保留 computation-balanced partitioning，再用 D2D swap、GPU-CPU swap 和 recomputation 将高显存压力 stage 的张量转移到轻负载 GPU 或主机侧。

## 实验结果

- 相对 ZeRO：相同显存约束下 **1.7× – 2.3×** 加速
- DGX-1 → DGX-2 近线性扩展
