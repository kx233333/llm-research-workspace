# GB200-NVL72

> NVIDIA 的整机柜超节点：**72 GPU 单一 NVLink 域**，消除了传统 8 卡节点外的 IB 带宽悬崖，是 [[DWDP]]、[[MegaScale-Infer]] 等 a2a 优化方案的硬件前提。

## 关键规格

| 项目 | 数值 |
|------|------|
| GPU | 72× B200 SXM |
| GPU 显存 | 192 GB HBM3e × 72 = 13.8 TB |
| CPU | 36× Grace（ARM Neoverse V2） |
| CPU 内存 | 480 GB LPDDR5X × 36 = 17.3 TB |
| **NVSwitch 芯片** | **18 个（9 托盘，每盘 2 芯片）** |
| NVLink 版本 | 5.0（1.8 TB/s per GPU bidirectional） |
| NVLink 域内聚合带宽 | **130 TB/s** |
| 形态 | 整机柜，液冷，~125 kW |
| 节点间 NIC | ConnectX-8（800 Gbps） |

## 五层带宽（见 [[2026-04-22-gpu-bandwidth-hierarchy]]）

| 层级 | 带宽 | 对比 DGX H100 |
|------|------|-------------|
| L1 HBM | 8 TB/s | 2.4× |
| L2 CPU DRAM | 546 GB/s/CPU × 36 | — |
| L3 CPU↔GPU | **900 GB/s NVLink-C2C** | **~7×** |
| L4 GPU 间（NVLink 5） | 1.8 TB/s / GPU | 2× |
| L4 聚合 | **130 TB/s（72 GPU）** | **18×** |
| L5 节点间 | 800 Gbps | 2× |

## 对算法设计的三大突破

1. **把 NVLink 从节点内扩展到机柜级**：消除 8 卡以上规模的 IB 瓶颈
2. **NVLink-C2C 打通显存墙与系统内存**：CPU DRAM 可被 GPU 以近显存速度访问，形成 ~30 TB 统一内存池
3. **P2P 带宽足以隐藏 MoE expert 预取**：[[DWDP]] 才得以成立

## 谁需要这个硬件

- [[DWDP]]：需 1.8 TB/s per GPU P2P + 单 NVLink 域
- [[MegaScale-Infer]]：需高 M2N 带宽
- 跨 8 卡规模的任何 MoE 训练/推理

## 谁不需要（在其他硬件也能跑）

- [[MPress]]（单节点 DGX-1/2 即可）
- [[ChunkFlow]]（常规 GPU 集群）
- [[WLB-LLM]]、[[ByteScale]]（主要跑在 H100 集群）
