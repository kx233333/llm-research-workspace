# 分离部署

> **Disaggregated Serving**：把推理的不同阶段或模块放到不同的资源池，异构部署。近年 LLM 推理的主流设计范式。

## 三种常见分离维度

### 1. Prefill / Decode 分离

- Prefill 是 compute-bound 短任务
- Decode 是 memory-bound 长任务
- 代表：DistServe、Splitwise、Mooncake

### 2. [[Attention-FFN 解耦]]

- Attention memory-bound，FFN compute-bound
- 代表：[[MegaScale-Infer]]、[[AFD-Ratio 理论]]

### 3. Expert 分布

- [[DWDP]]：expert 权重分散存，按需 P2P 拉取

## 关键收益

1. **异构硬件匹配**：各模块用最划算的 GPU
2. **独立扩展**：流量变化时可独立 scale 某一模块
3. **消除跨模块耦合**：避免 worst-of-both 场景

## 关键代价

1. 模块间通信开销（需高带宽互联）
2. 调度复杂度
3. Profiling 需求增加
