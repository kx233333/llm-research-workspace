# 长上下文训练

> 训练阶段支持 32K–2M tokens 上下文的技术总称。核心挑战是 Attention 的 $O(s^2)$ 计算和 KV 激活显存。

## 主要子方向

### 1. Context Parallelism (CP)

- DeepSpeed-Ulysses（a2a）
- Ring-Attention（p2p）
- [[USP]]（Hybrid）

### 2. 变长序列均衡

- [[WLB-LLM]]：var-length packing + outlier delay
- [[ByteScale]]：HDP + balance scheduler
- [[LobRA]]：异构 replica
- [[FlexSP]]：异构 SP 组
- [[ChunkFlow]]：chunk 粒度

### 3. 核心注意力解耦

- [[CAD-DistCA]]：DP 场景下的彻底解耦

## 关键数据特性

- **长尾分布**：ChunkFlow 测得 >99% 序列短于 4K，最长可达 303K
- **混合长短训练的必要性**：提升 short-context 性能

## 长上下文下的瓶颈位置

```
序列长度:   4K      8K      32K     128K    1M
─────────────────────────────────────────────
主要瓶颈:   FFN     混合    Attn    Attn    Attn
                   $s^2$ 项开始占优  显存 + 计算双爆炸
```
