# 算子间并行

> Inter-Operator Parallelism，即把模型的算子（层）按顺序划分到不同 GPU，每个 GPU 只持有部分层。对应工业界术语 **Pipeline Parallelism (PP)**。

## 关键特性

- 与 ZeRO 相比：显存冗余更低，cross-GPU 通信量更小
- 对 Transformer 类深层顺序结构天然友好
- 典型框架：PipeDream、DAPPLE、Megatron-LM、GPipe

## 核心问题

- **Pipeline Bubble**：stage 间等待空隙
- **Memory Skew**：前几个 stage 需保存更多激活
- **Stage 划分不均**：长序列下 Attention 层和非 Attention 层 FLOPs 差异大

## 缓解方案

- [[State-Aware 1F1B]]（配合 [[ChunkFlow]]）
- [[MPress]]：D2D swap 平衡各 stage 显存
- BPipe：memory-balanced partition
