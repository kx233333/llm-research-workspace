# 显存优化

> GPU 显存墙的系统级应对。三条主线：**swap**（借用其他存储）、**recompute**（用计算换显存）、**sharding**（按结构分片）。

## 三条主线对比

| 主线 | 代表 | 机制 | 代价 |
|------|------|------|------|
| **Swap to CPU** | ZeRO-Offload | 激活/权重搬到 CPU DRAM | PCIe 16 GB/s，极慢 |
| **Swap to peer GPU** | [[MPress]] | D2D NVLink swap | NVLink 600+ GB/s，快 |
| **Swap to Superchip** | [[SuperOffload]] | C2C 直连 CPU | C2C 900 GB/s，更快 |
| **Recompute** | Activation Recomputation | backward 时重算前向 | 2× 计算开销 |
| **Sharding** | ZeRO-1/2/3, FSDP | 按 DP 分参数/梯度/优化器 | 通信开销 |
| **Chunk-level 显存重构** | [[ChunkFlow]] | chunk 串行 + 选择性重算 | 少量额外 forward |

## 核心观点

1. **FlashAttention 已解决 Attention 激活 $O(s^2)$ 问题**：训练侧主要显存瓶颈转到 FFN activations 和优化器状态。
2. **Swap 目标从 CPU 转到 peer GPU/Superchip 是近期趋势**：带宽升两个量级。
3. **[[ChunkFlow]] 的路径：用 chunksize 控制显存上限**，完全与最大序列长度解耦。
4. **KV Cache 是推理侧主要显存消耗者**，需专门的压缩/offload 策略。
