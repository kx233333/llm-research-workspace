# 超节点架构

> 一类新的 GPU 集群组织方式：把"NVLink 域"从节点内（8 卡）扩展到机柜级（72+ 卡）或 Superchip 级（CPU-GPU 紧耦合）。典型代表 [[GB200-NVL72]] 和 [[GH200-Superchip]]。

## 核心观点

1. **NVLink 域扩展突破了传统 8 卡边界**：72 GPU 单 NVLink 域让 a2a / EP 在更大范围内可行。
2. **C2C 紧耦合重塑 CPU-GPU 关系**：CPU DRAM 变成 GPU 可用的 L3 存储。
3. **硬件驱动算法**：[[DWDP]]、[[MegaScale-Infer]]、[[SuperOffload]] 都是超节点原生的方案。
4. **硬件也重设限**：ChunkSize、TP 度、Ulysses 度 的上限都被重新讨论（[[USP]] 的开放问题）。

## 对算法设计的三个根本改变

1. **a2a 的带宽不再是瓶颈**（单 NVLink 域内）
2. **CPU DRAM 成为 GPU 的有效存储层**
3. **跨节点 IB 仍然是边界**（跨 NVL72 机柜仍需 800 Gbps ConnectX-8）

## 未解决的问题

1. 超节点下 TP × EP × Ulysses 的三维空间如何划分？
2. ChunkFlow 类串行能扩展到 72 GPU 域吗？
3. 跨超节点的通信拓扑（多 NVL72 机柜）？
4. Superchip 的 CPU 是否承担更多"分布式训练辅助角色"（如 optimizer step、router）？
