# 负载均衡与变长序列

> 变长序列训练/推理中多 GPU 的计算、通信、显存负载均衡问题。**Attention $\propto s^2$ vs 其他 $\propto s$ 是根本矛盾**，由此衍生多条解决路径。

## 核心观点

1. **根本矛盾**：Attention FLOPs $\propto s^2 h$，GEMM/通信/element-wise $\propto sh^2$（linear in $s$）。变长序列下 $\sum l_i^2$ 与 $\sum l_i$ 无法同时平衡。
2. **FlashAttention 改变了显存侧的局面**：Attention 激活从 $O(s^2)$ 降到 $O(s)$，**显存不平衡大幅缓解，但计算不平衡依然存在**。
3. **KV Cache 是推理侧专属难题**：随上下文线性增长、随 batch 线性增长，Attention 独有。
4. **解决路径分叉**：
   - 估算改进（WLB-LLM）
   - 数据重组（ChunkFlow）
   - 配置异构（FlexSP、LobRA）
   - 破坏约束（ByteScale 打破 micro-batch 等量）
   - 显式解耦（CAD / MegaScale-Infer）
5. **a2a 同步是训练/推理对不均衡最敏感的放大器**：优化 a2a 或消灭 a2a 是多数方案的共同动作。
