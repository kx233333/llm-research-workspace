# 并行策略与同步开销

> DP / TP / PP / CP / EP 各自的同步机制及其开销。**集合通信的屏障语义是所有不均衡敏感性的根源**。

## 核心观点

1. **同步屏障是不均衡敏感的数学根源**：任何集合通信都"等最慢的"。
2. **a2a 是最敏感的同步点**（MoE / CP），因为它不对称 + 高频（每层 2 次）+ 同时受计算和通信不均衡影响。
3. **绕开 a2a 有三条路**：
   - 消灭（[[DWDP]]、[[ChunkFlow]] 的 CP 类串行）
   - 隐藏（[[MegaScale-Infer]] ping-pong、DualPipe）
   - 源头减少不均衡（routing-level balancing）
4. **NVL72 硬件把 a2a 从瓶颈变非瓶颈，但不消除同步语义**。

## 各并行策略的同步点

| 策略 | 关键同步 | 频率 |
|------|---------|------|
| DP | all-reduce（梯度聚合） | 每 iter 1 次 |
| TP | all-reduce / all-gather / reduce-scatter | 每层 2 次 |
| PP | P2P + 最终 all-reduce | 每 micro-batch |
| CP | a2a（Ulysses）/ Ring P2P | 每 Attention 层 |
| EP | a2a（dispatch + combine） | 每 MoE 层 × 2 |
| ZeRO-3 / FSDP | all-gather + reduce-scatter | 每层 |

## 素材汇总

| 素材 | 关注的同步类型 | 应对方法 |
|------|------------|---------|
| DWDP | MoE EP a2a | 异步 P2P 预取 |
| ChunkFlow | CP a2a + PP bubble | chunk 串行 + state-aware |
| MPress | PP 显存不均 | D2D NVLink swap |
| WLB-LLM | PP + CP 计算不均 | var-len packing |
| MegaScale-Infer | MoE a2a | ping-pong overlap |

## 跨素材洞察

**docx 调研者的核心猜想**：
> 解决负载均衡的有效方法是优化掉 a2a 操作，那么 CP 中的 a2a 如何被优化掉，改成 ChunkFlow 那样的类串行？

这把 DWDP（消灭 EP a2a）和 ChunkFlow（消灭 CP a2a）统一到同一个框架下，是值得深入的研究方向。<!-- confidence: INFERRED -->
